**setup-wizard.tsx (container):**
- Import and use `useSetupWizard` hook from `@/hooks/useSetupWizard`
- Import `useDataPersistence` from `@/hooks/useDataPersistence` for import functionality
- Props: `{ onComplete: () => void }`
- State: errors state `Record<string, string>` for step validation display
- Step rendering logic:
  - Step 0: `<WizardStepWelcome onStartWizard={() => goNext()} onImport={handleImport} />`
  - Step 1: `<WizardStepBalance data={wizardData} onChange={setWizardData} onNext={handleNext} errors={errors} />`
  - Step 2: `<WizardStepUsd data={wizardData} onChange={setWizardData} onNext={handleNext} onBack={goBack} onSkip={handleSkip} errors={errors} />`
  - Step 3: `<WizardStepIncome data={wizardData} onChange={setWizardData} onNext={handleNext} onBack={goBack} onSkip={handleSkip} errors={errors} />`
  - Step 4: `<WizardStepSummary data={wizardData} onConfirm={handleConfirm} onEdit={setCurrentStep} />`

- `handleNext`: call `validateCurrentStep()`, if errors non-empty set error state, else clear errors and call `goNext()`
- `handleSkip`: clear errors, call `goNext()` (skip doesn't validate)
- `handleConfirm`: call `commit()`, then `window.location.reload()` to remount everything cleanly (same pattern as import)
- `handleImport`: Create hidden file input ref, trigger click, on file change call `importData(file)`. importData already calls `window.location.reload()` on success. Show alert on error.
- Progress indicator: Show "Paso {currentStep} de 4" when currentStep >= 1 (not shown on welcome step 0). Use a simple div with 4 circles/dots, highlighting current step. Keep it minimal — no external stepper library.
- Layout: Center the wizard card on screen with `flex items-center justify-center min-h-screen`. Max width ~md (max-w-md). Use the same background as the app (the gradient in page.tsx will still show behind).

**2. Inside the ExpenseTracker function, add wizard detection state BEFORE all existing hooks:**

```typescript
const isHydrated = useHydration();
const [showWizard, setShowWizard] = useState(false);

useEffect(() => {
  if (isHydrated) {
    const hasMonthlyData = localStorage.getItem("monthlyData");
    const hasSalaryHistory = localStorage.getItem("salaryHistory");
    setShowWizard(!hasMonthlyData && !hasSalaryHistory);
  }
}, [isHydrated]);
```

**3. Add early returns BEFORE the main JSX return:**

```typescript
if (!isHydrated) return null;  // Prevent SSR mismatch — render nothing until hydrated

if (showWizard) {
  return (
    <SetupWizard
      onComplete={() => window.location.reload()}
    />
  );
}
```

**Important:** The `return null` for pre-hydration must come before ANY rendering. The wizard check comes right after. Only if both pass does the normal tabbed UI render.

**4. Do NOT modify page.tsx.** The wizard uses `window.location.reload()` on completion (same pattern as JSON import), which is simpler than the key-based remount and avoids converting page.tsx to a client component.

**5. Do NOT modify any existing hooks.** The wizard gate is purely a conditional render check in ExpenseTracker.
