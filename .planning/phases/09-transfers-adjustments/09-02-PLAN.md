**1. Create `components/transfer-dialog.tsx`** — Unified "Nuevo movimiento" dialog with type selector and dynamic fields.

Follow the UsdPurchaseDialog pattern (Radix Dialog, controlled open state, form submission).

Props interface:
```typescript
interface TransferDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  onAddTransfer: (data: Omit<Transfer, "id" | "createdAt">) => void;
  defaultDate: string;  // yyyy-MM-dd
  globalUsdRate: number; // For pre-filling the rate on currency transfers
}
```

Implementation details:
- **Type selector**: Use Radix Select with 4 options:
  - "Transferencia ARS → USD" (value: "currency_ars_to_usd")
  - "Transferencia USD → ARS" (value: "currency_usd_to_ars")
  - "Retiro a efectivo" (value: "cash_out")
  - "Deposito desde efectivo" (value: "cash_in")
- **State**: `transferType` (default "currency_ars_to_usd"), `arsAmount`, `usdAmount`, `amount`, `currency` ("ARS" | "USD"), `note`, `date`
- **Dynamic fields by type:**
  - `currency_ars_to_usd` / `currency_usd_to_ars`: Show ARS amount input + USD amount input. Auto-calculate effective rate display: `arsAmount / usdAmount`. Pre-fill rate from `globalUsdRate`. Same UX as current Compra USD dialog — user enters both amounts, rate shown as read-only derived value.
  - `cash_out`: Show amount input + currency toggle (ARS/USD two-button, like employment type toggle)
  - `cash_in`: Show amount input + currency toggle + optional text note input (placeholder: "Origen del efectivo")
- **Date field**: date input defaulting to `defaultDate`
- **Effective rate display**: For currency types, show below the amount inputs: "Cotizacion efectiva: $X" (arsAmount / usdAmount, or "---" if either is 0)
- **Submit handler**: Build Transfer data object based on type, call `onAddTransfer`, close dialog, reset form
- **Validation**: Amount(s) must be > 0. For currency transfers, both arsAmount and usdAmount must be > 0.
- **Reset form on close**: Clear all fields when dialog closes

Icons: Use `ArrowLeftRight` from lucide-react for the dialog header.

  - `cash_out`: "Retiro a efectivo ({currency})"
  - `cash_in`: description field if present, else "Deposito desde efectivo ({currency})"
  - `adjustment_ars`: "Ajuste saldo ARS"
  - `adjustment_usd`: "Ajuste saldo USD"
- **Monto column**: Use FormattedAmount component. For currency transfers show both sides: "-$X / +US$Y" (or vice versa). For cash/adjustment show single amount with currency.
- **Acciones column**: Trash2 icon button to delete (same pattern as expenses table)
- **Empty state**: "No hay movimientos en este periodo" centered text when transfers is empty
- **Date formatting**: Use format from date-fns with "dd/MM/yyyy" pattern
