Create `components/adjustment-dialog.tsx` — two-step dialog for balance adjustments.

Props interface:
```typescript
interface AdjustmentDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  onCreateAdjustment: (currency: "ARS" | "USD", realBalance: number, trackedBalance: number) => void;
  arsBalance: number;   // Current tracked ARS balance
  usdBalance: number;   // Current tracked USD balance
}
```

Implementation details:

**State management:**
```typescript
const [step, setStep] = useState<"input" | "confirm">("input");
const [currency, setCurrency] = useState<"ARS" | "USD">("ARS");
const [realBalance, setRealBalance] = useState<string>("");
```

**Step 1 (input):**
- Dialog title: "Ajustar saldo real" with Scale icon from lucide-react
- Currency selector: Two-button toggle (ARS / USD), same style as employment type toggle in config-card
- Current tracked balance display (read-only): "Saldo segun la app:" followed by FormattedAmount showing `currency === "ARS" ? arsBalance : usdBalance`
- Real balance input: Number input with label "Tu saldo real:" and currency prefix ($ or US$)
- "Siguiente" button to go to step 2 — disabled if realBalance is empty or not a valid number
- Cancel button to close dialog

**Step 2 (confirm):**
- Dialog title: "Confirmar ajuste"
- Math display with clear visual layout:
  ```
  Saldo segun app:  $145.000  (or US$ X)
  Saldo real:       $150.000  (or US$ X)
  ─────────────────────────────
  Ajuste:           +$5.000   (green if positive, red if negative)
  ```
- Use FormattedAmount for all amounts. The adjustment amount = realBalance - trackedBalance
- If adjustment is 0, show "No se necesita ajuste" message and disable confirm button
- "Confirmar ajuste" button (primary) — calls `onCreateAdjustment(currency, parseFloat(realBalance), trackedBalance)`, then closes dialog
- "Volver" button to go back to step 1

**CRITICAL: Read balance at confirmation time.** The `arsBalance` and `usdBalance` props will be current because React re-renders when state changes. The tracked balance used for the math display in step 2 should be read from props at render time (`currency === "ARS" ? arsBalance : usdBalance`), NOT captured in state at step transition.

**Reset on close:** When dialog closes (via onOpenChange), reset step to "input", clear realBalance, reset currency to "ARS".

**Styling:**
- Math display section: Use a bordered container (border rounded-lg p-4)
- Amounts right-aligned in a grid or flex layout
- Adjustment line uses `text-green-600` for positive, `text-red-600` for negative
- Separator: `<hr className="my-2 border-border" />`
