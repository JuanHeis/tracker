Create `hooks/useBudgetTracker.ts` with the following:

**Data model:**
```typescript
interface BudgetDefinition {
  category: Category;
  monthlyLimit: number; // ARS
}

interface BudgetSnapshot {
  [category: string]: number; // category -> limit at that time
}

interface BudgetData {
  definitions: BudgetDefinition[];
  snapshots: Record<string, BudgetSnapshot>; // monthKey -> snapshot
}

interface BudgetProgress {
  category: Category;
  limit: number;          // ARS limit for this month
  spent: number;          // ARS spent in period
  percentage: number;     // spent / limit * 100
  expenses: Array<{ id: string; name: string; amount: number; arsAmount: number }>; // for tooltip breakdown
}
```

**Hook signature:**
```typescript
export function useBudgetTracker(
  expenses: Expense[],
  monthKey: string,
  viewMode: ViewMode,
  payDay: number
): {
  budgetData: BudgetData;
  budgetProgress: BudgetProgress[];  // sorted by percentage desc, exceeded first
  totalBudgeted: number;
  totalSpent: number;
  addBudget: (category: Category, monthlyLimit: number) => void;
  updateBudget: (category: Category, monthlyLimit: number) => void;
  deleteBudget: (category: Category) => void;
  categoriesWithoutBudget: Category[]; // for the "add budget" dropdown
}
```

**Implementation details:**

1. Use `useLocalStorage<BudgetData>("budgetData", { definitions: [], snapshots: {} })` for persistence.

2. **addBudget:** Push new BudgetDefinition. Reject if category already has a budget.

3. **updateBudget:** Find definition by category, update monthlyLimit. Do NOT touch existing snapshots (they preserve historical limits).

4. **deleteBudget:** Remove definition by category. Do NOT delete historical snapshots.

5. **Spending computation** (in `useMemo`):
   - Get date range via `getFilterDateRange(monthKey, viewMode, payDay)`
   - Filter expenses by date range (parse with `parse(expense.date, "yyyy-MM-dd", new Date())`)
   - For each expense in a budgeted category, convert to ARS: `expense.currencyType === CurrencyType.USD ? expense.amount * expense.usdRate : expense.amount`
   - Group by category, sum ARS amounts
   - Build `BudgetProgress[]` for each budgeted category

6. **Snapshot logic:**
   - When computing progress for a month: check if `snapshots[monthKey]` exists
   - If snapshot exists: use `snapshots[monthKey][category]` as the limit
   - If no snapshot AND the month has expenses in any budgeted category: create snapshot by capturing current definition limits, persist it
   - If no snapshot AND no expenses: use current definition limits (don't create empty snapshots)
   - This is the "lazy snapshot on first computation" pattern from CONTEXT.md

7. **categoriesWithoutBudget:** All Category values minus those in definitions.

8. **Sorting:** `budgetProgress` sorted by percentage descending. Exceeded (>= 100%) always at top.

9. **totalBudgeted / totalSpent:** Sum across all budget progress entries.

Import `Category`, `Expense`, `CurrencyType` from `@/hooks/useMoneyTracker`. Import `getFilterDateRange`, `ViewMode` from `@/hooks/usePayPeriod`. Import `useLocalStorage` from `@/hooks/useLocalStorage`. Import `parse` from `date-fns`.
