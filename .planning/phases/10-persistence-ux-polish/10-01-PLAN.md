  <name>Task 1: Create useDataPersistence hook with export/import/validate</name>
  <files>hooks/useDataPersistence.ts</files>
  <action>
Create `hooks/useDataPersistence.ts` with:

1. **STORAGE_KEYS constant** enumerating all 7 localStorage keys:
   - `monthlyData`, `globalUsdRate`, `salaryHistory`, `incomeConfig`, `recurringExpenses`, `budgetData`, `lastUsedUsdRate`

2. **ExportEnvelope interface:**
   ```typescript
   interface ExportEnvelope {
     appName: "expense-tracker";
     exportVersion: 1;
     exportDate: string; // ISO timestamp
     data: Record<string, unknown>; // all 7 keys
   }
   ```

3. **exportData() function:**
   - Reads ALL 7 localStorage keys
   - For JSON-stored keys (monthlyData, salaryHistory, incomeConfig, recurringExpenses, budgetData): parse with JSON.parse
   - For numeric keys (globalUsdRate, lastUsedUsdRate): parse with parseFloat, default to 0 if missing
   - Wraps in ExportEnvelope with appName, exportVersion: 1, exportDate: new Date().toISOString()
   - Creates Blob with JSON.stringify(envelope, null, 2), type "application/json"
   - Creates anchor element, sets href to URL.createObjectURL(blob), download to `expense-tracker-backup-${format(new Date(), "yyyy-MM-dd")}.json` using date-fns format
   - Clicks anchor, revokes object URL

4. **validateEnvelope(data: unknown) function** returning `{ valid: true; envelope: ExportEnvelope } | { valid: false; error: string }`:
   - Check data is object with appName === "expense-tracker"
   - Check exportVersion is number
   - Check data.data is object
   - Check all 7 required keys exist in data.data
   - Return descriptive error message if any check fails

5. **importData(file: File) function** returning `Promise<{ success: true } | { success: false; error: string }>`:
   - Read file with FileReader.readAsText
   - JSON.parse the result (catch parse errors -> "El archivo no es un JSON valido")
   - Call validateEnvelope (return error if invalid)
   - Write each key to localStorage:
     - JSON keys: JSON.stringify before setItem
     - Numeric keys: String() before setItem
   - After writing all keys, call `window.location.reload()` to reset all React state
   - Note: migrateData() will run automatically on reload since useMoneyTracker calls it on init

6. **useDataPersistence() hook** returning `{ exportData, importData }` for component consumption.

Use date-fns `format` for filename date (already in project). No new dependencies.
