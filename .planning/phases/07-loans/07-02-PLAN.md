**1. Create `components/loan-dialog.tsx`** -- Loan creation dialog with Preste/Debo mode toggle.

Design decisions:
- **Mode toggle:** Two-button toggle (like employment type in salary card, NOT a dropdown) -- one button for "Preste" (green accent), one for "Debo" (red accent). Active state uses filled background.
- **Fields:** Persona (text input, required), Monto (number input, required, > 0), Moneda (two-button toggle ARS/USD), Fecha (date input, default today), Nota (textarea, optional)
- **Dialog title:** Changes with mode -- "Registrar prestamo" for Preste, "Registrar deuda" for Debo
- **Submit button:** "Registrar prestamo" / "Registrar deuda" matching the mode
- **Validation:** Persona required (trimmed non-empty), Monto > 0. Show toast or disable submit if invalid.
- **On submit:** calls onAddLoan callback with form data, closes dialog, resets form

Props interface:
```typescript
interface LoanDialogProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  onAddLoan: (data: {
    type: LoanType;
    persona: string;
    amount: number;
    currencyType: CurrencyType;
    date: string;
    note?: string;
  }) => void;
  defaultDate: string;
}
```

Use shadcn Dialog, Input, Button, Label components. Import LoanType from useMoneyTracker, CurrencyType from constants/investments.

**2. Create `components/loan-payments.tsx`** -- Payment history list + inline add payment form.

Follows the investment-movements.tsx pattern (expandable content with history + inline form).

Design decisions:
- **Payment list:** Chronological list showing date (formatted with date-fns/es), amount (using FormattedAmount or currencySymbol + toLocaleString), newest first
- **Inline form:** Single row with date input (default today) + amount input + "Registrar" button
- **Amount validation:** Must be > 0 and <= remaining balance. Show remaining balance above the form: "Resta: $X"
- **Empty state:** "Sin pagos registrados" text when no payments exist
- **Currency display:** Uses the loan's currencyType (passed as prop) for all amounts

Props interface:
```typescript
interface LoanPaymentsProps {
  loan: Loan;
  onAddPayment: (loanId: string, data: { date: string; amount: number }) => void;
}
```

Compute remaining inside the component: `loan.amount - loan.payments.reduce((sum, p) => sum + p.amount, 0)`. If remaining <= 0 (loan fully paid), hide the add payment form.

**1. Create `components/loan-row.tsx`** -- Single loan row with expand/collapse.

Follows the investment-row.tsx pattern.

Design decisions:
- **Row click:** Toggles expand/collapse to show LoanPayments
- **Columns:** Persona, Tipo (Badge: green "Preste" / red "Debo"), Monto original (FormattedAmount or formatted number), Resta (computed: amount - sum(payments), shown in loan currency), Moneda (ARS/USD text), Fecha (formatted dd/MM/yyyy), Estado (Badge: colored by status), Acciones (edit pencil, delete trash, chevron)
- **Dimmed style:** Completed loans (status Cobrado, Pagado, or Perdonado) get `opacity-60` class like finalized investments
- **Status badge colors:** Pendiente = yellow/amber, Cobrado = green, Pagado = green, Perdonado = gray/muted
- **Tipo badge colors:** Preste = green background, Debo = red background
- **Expanded content:** Renders `<LoanPayments>` component in a full-width TableCell with `bg-muted/30 p-4`
- **Action buttons:** Pencil icon (always visible, muted color + hover:blue) for edit, Trash icon (muted + hover:red) for delete. For "preste" loans with status "Pendiente", add a "Perdonar" action (can be a small text button or icon in expanded row)
