Create `hooks/useProjectionEngine.ts` — the orchestrator hook for all projections.

**File structure:**
```
"use client";
imports
hook function
  - parameters with defaults
  - read data from existing hooks
  - compute projections via useMemo
  - build chart-ready output
  - return UseProjectionEngineReturn
```

**Hook signature:**
```typescript
export function useProjectionEngine(params?: {
  horizonMonths?: number;         // Default: 12
  includeContributions?: boolean; // Default: false (PROJ-02: toggled by UI)
}): UseProjectionEngineReturn
```

**Implementation steps (all inside a single useMemo):**

1. **Read data from props/parameters:**
   The hook needs monthlyData, globalUsdRate, salaryHistory, recurringExpenses. However, since this is a standalone hook, it should receive these as parameters rather than calling other hooks (to avoid tight coupling and allow the parent component to pass the data). BUT the project convention is hooks calling hooks (useMoneyTracker calls sub-hooks). 

   **Decision: Accept data as parameters** to keep the hook pure-ish and testable:
   ```typescript
   export function useProjectionEngine(
     monthlyData: MonthlyData,
     salaryEntries: SalaryEntry[],
     recurringExpenses: RecurringExpense[],
     globalUsdRate: number,
     options?: {
       horizonMonths?: number;
       includeContributions?: boolean;
     }
   ): UseProjectionEngineReturn
   ```

   This way the parent component (Phase 16) passes data from existing hooks and the projection engine stays decoupled.

2. **Determine current month:** `format(new Date(), "yyyy-MM")` using date-fns

3. **Compute historical patrimony:**
   ```typescript
   const historical = reconstructHistoricalPatrimony(monthlyData, salaryEntries, globalUsdRate);
   ```

4. **Compute current patrimony** from the last historical point, or 0 if empty:
   ```typescript
   const currentPatrimony = historical.length > 0 ? historical[historical.length - 1].patrimony : 0;
   ```

5. **Compute current salary** using getSalaryForMonth:
   ```typescript
   const currentSalary = getSalaryForMonth(currentMonth, salaryEntries, monthlyData.salaryOverrides || {}).amount;
   ```

6. **Compute monthly net savings (pass globalUsdRate for USD recurring expense conversion):**
   ```typescript
   const netSavings = estimateMonthlyNetSavings(currentSalary, recurringExpenses, globalUsdRate);
   ```

6b. **Compute income projection (PROJ-03):**
   ```typescript
   const incomeProjection = projectIncome(currentSalary, horizonMonths);
   ```
   This flat-line array is exposed in the return object so Phase 16 can render the income series.

7. **Compute investment projections** (base scenario):
   - Filter active, non-liquid investments: `monthlyData.investments.filter(i => i.status === "Activa" && !i.isLiquid)`
   - For each investment:
     - If Plazo Fijo with tna: `monthlyRate = pfMonthlyRate(investment.tna)`
     - Else: `monthlyRate = getDefaultMonthlyRate(investment.type)`
     - `projectInvestment(investment, monthlyRate, horizonMonths, includeContributions)`
   - Store as `investmentProjections` array

8. **Compute total investment growth per month** (for patrimony projection):
   - For each month m (0..horizonMonths):
     - Sum all investmentProjection.projectedValues[m] across all investments
     - Convert USD investments to ARS: if investment.currencyType === USD, multiply by globalUsdRate
   - `investmentGrowthByMonth[m]` = total at month m - total at month 0 (growth only)

9. **Compute patrimony scenarios:**
   ```typescript
   const scenarioBase = projectPatrimonyScenarios(currentPatrimony, netSavings, horizonMonths);
   ```
   Then add investment growth to each scenario:
   - For optimista: recompute investment projections with rateMultiplier 1.5, then add growth
   - For base: use base investment growth
   - For pesimista: recompute investment projections with rateMultiplier 0.5, then add growth

   **Implementation:** Compute investment growth for each scenario separately:
   ```typescript
   function computeInvestmentGrowth(investments, rateMultiplier, horizonMonths, includeContributions, globalUsdRate): number[] {
     const projections = investments.map(inv => {
       const rate = inv.type === "Plazo Fijo" && inv.tna
         ? pfMonthlyRate(inv.tna) * rateMultiplier
         : getDefaultMonthlyRate(inv.type, rateMultiplier);
       return projectInvestment(inv, rate, horizonMonths, includeContributions);
     });
     // Sum projected values per month, converting USD to ARS
     const growth: number[] = [];
     for (let m = 0; m <= horizonMonths; m++) {
       let total = 0;
       for (const p of projections) {
         const value = p.projectedValues[m] || 0;
         total += p.currencyType === CurrencyType.USD ? value * globalUsdRate : value;
       }
       growth.push(total - totalAtMonth0);  // Growth only
     }
     return growth;
   }
   ```

10. **Combine scenario savings + investment growth:**
    For each scenario s and month m:
    `final[s][m] = scenarioBase[s][m] + investmentGrowth[s][m]`

11. **Build ProjectionDataPoint[] for chart:**
    - Historical months: from historical array, mapped to ProjectionDataPoint with `historicalPatrimony` set, all projection fields null
    - Current month: both `historicalPatrimony` AND `proyeccionBase` set (overlap point)
    - Future months: generate month labels using `addMonths` + `format(date, "MMM yy")`, set projection fields, historicalPatrimony null
    - Month labels in Spanish: use `{ locale: es }` from date-fns — import `es` from `date-fns/locale/es`
    - Capitalize first letter of month name for display

12. **Compute currentMonthIndex:** index in patrimonyData where monthKey === currentMonth

13. **Compute projectedPatrimony:** final values at horizonMonths for each scenario

14. **Return UseProjectionEngineReturn** with all computed values, including `incomeProjection` array (PROJ-03)

**Wrap entire computation in `useMemo`** with dependencies: `[monthlyData, salaryEntries, recurringExpenses, globalUsdRate, horizonMonths, includeContributions]`

**Edge cases:**
- No investments → investmentProjections = [], investment growth = 0 for all months
- No salary → netSavings = 0, scenarios grow only by investment growth
- No history → historical = [], currentPatrimony = 0
- globalUsdRate = 0 → USD values contribute 0 (avoid NaN)

**DO NOT:**
- Call useLocalStorage, useMoneyTracker, useSalaryHistory, or useRecurringExpenses from inside this hook — data comes via parameters
- Write to localStorage (INFRA-03)
- Use useState — this hook is pure computation, useMemo only
