**Functions to implement:**

1. `pfMonthlyRate(tnaPercent: number): number`
   - Converts TNA (Tasa Nominal Anual) to monthly effective rate
   - Formula: `Math.pow(1 + tnaPercent / 100 / 365, 30) - 1`
   - Argentine PFs compound daily, this approximates monthly

2. `getDefaultMonthlyRate(type: InvestmentType, rateMultiplier: number = 1): number`
   - For PF: returns 0 (caller must use pfMonthlyRate with investment.tna)
   - For others: `DEFAULT_ANNUAL_RATES[type] * rateMultiplier / 12`
   - Import DEFAULT_ANNUAL_RATES from types.ts

3. `projectInvestment(investment: Investment, monthlyRate: number, horizonMonths: number, includeContributions: boolean): InvestmentProjection`
   - Start with `investment.currentValue` as month 0
   - For each month 1..horizonMonths: `balance = (balance + contribution) * (1 + monthlyRate)`
   - Round each value to integer with `Math.round()`
   - If `includeContributions` is true, find last non-initial aporte: `investment.movements.filter(m => m.type === "aporte" && !m.isInitial).at(-1)?.amount ?? 0`
   - If false, monthlyContribution = 0
   - Return InvestmentProjection with all fields populated

**Edge cases to handle:**
- Investment with currentValue 0 → valid, projects from 0
- Investment with no movements → monthlyContribution = 0
- Investment with only isInitial movements → monthlyContribution = 0 (wizard patrimony excluded per RESEARCH notes)
- monthlyRate = 0 → balance grows only by contributions
- horizonMonths = 0 → return array with just currentValue

Import Investment type from `@/hooks/useMoneyTracker`. Import InvestmentProjection, DEFAULT_ANNUAL_RATES from `./types`.

1. `projectIncome(currentSalary: number, horizonMonths: number): number[]`
   - Returns array of length `horizonMonths + 1` (index 0 = current month)
   - All values = currentSalary (flat line projection per PROJ-03)

2. `estimateMonthlyNetSavings(currentSalary: number, activeRecurringExpenses: RecurringExpense[], globalUsdRate: number): number`
   - Sum amounts of recurring expenses where `status === "Activa"`:
     - ARS expenses: add `r.amount` directly
     - USD expenses: add `r.amount * globalUsdRate` (converts to ARS for net savings calculation)
   - Return `Math.max(0, currentSalary - totalRecurring)` — never negative
   - Import RecurringExpense from `@/hooks/useRecurringExpenses`, CurrencyType from `@/constants/investments`

**File 2: `lib/projection/patrimony-history.ts`**

1. `reconstructHistoricalPatrimony(monthlyData: MonthlyData, salaryHistory: SalaryEntry[], globalUsdRate: number): HistoricalPoint[]`
   - Collect all unique month keys (yyyy-MM) from: expenses dates, extraIncomes dates, investment movements dates, transfers dates, loan dates, salaryHistory effectiveDate entries
   - Sort chronologically
   - For each month, compute total patrimony:
     a. **Salary:** Use getSalaryForMonth(monthKey, salaryHistory, monthlyData.salaryOverrides || {}).amount
     b. **Extra incomes:** Sum extraIncomes in that month. ARS amounts directly, USD amounts * globalUsdRate
     c. **Expenses:** Subtract expenses in that month. ARS directly, USD * globalUsdRate
     d. **Investment values:** For each active investment, use the sum of movements (aportes - retiros) as historical proxy. For the LATEST month in the series, use currentValue instead. Convert USD investments using globalUsdRate.
     e. **Transfers:** Process transfers by type (currency conversions are neutral, cash_out reduces, cash_in increases, adjustments modify)
     f. **Loans:** preste reduces liquid (asset tracked separately), debo payments reduce liquid
     g. **USD purchases:** tracked purchases reduce ARS, all add USD
   - Compute RUNNING patrimony: liquid ARS + liquid USD * globalUsdRate + investment values + loan assets - loan liabilities
   - This is a SIMPLIFICATION of calculateDualBalances — use cumulative running totals, not scoped by view mode
   - Return sorted array of HistoricalPoint objects

   **CRITICAL simplification:** Since historical patrimony reconstruction is complex and calculateDualBalances is the authoritative logic, use a SIMPLER approach:
   - For each month, compute: running liquid balance (cumulative salary + incomes - expenses +/- investment movements +/- transfers +/- loans) + current investment values (using movement-based proxy for past months, currentValue for latest)
   - All USD amounts converted to ARS at globalUsdRate (single rate for all history — this is a known limitation with disclaimer in Phase 16)

   Import: MonthlyData from `@/hooks/useMoneyTracker`, SalaryEntry + getSalaryForMonth from `@/hooks/useSalaryHistory`, HistoricalPoint from `./types`, CurrencyType from `@/constants/investments`

**File 3: `lib/projection/scenario-engine.ts`**

1. `projectPatrimonyScenarios(currentPatrimony: number, monthlyNetSavings: number, investmentProjections: InvestmentProjection[], globalUsdRate: number, horizonMonths: number): { optimista: number[]; base: number[]; pesimista: number[] }`
   - For each scenario in SCENARIOS constant:
     a. Start with currentPatrimony at month 0
     b. For each future month (1..horizonMonths):
        - Add `monthlyNetSavings * scenario.savingsMultiplier`
        - Add total investment growth for that month (sum of all investmentProjections[month] - investmentProjections[month-1], applying rateMultiplier is already baked into the investment projections — BUT scenarios need SEPARATE investment projections)
     c. Actually simpler approach: For each scenario, project total patrimony as:
        `patrimony[m] = currentPatrimony + (monthlyNetSavings * savingsMultiplier * m) + totalInvestmentGrowth[m]`
        where totalInvestmentGrowth is the sum of all investment projected values at month m minus their current values

   - **Revised approach** (cleaner):
     The function receives `investmentProjections` already computed for the BASE scenario. For optimista/pesimista, it recomputes growth with the rate multiplier. But since investment projections are per-investment and scenario-specific, the HOOK should compute separate investment projections per scenario.

     **Simplest correct approach:** This function takes `baseNetSavings`, `horizonMonths`, and for each scenario:
     - `projected[m] = currentPatrimony + (baseNetSavings * savingsMultiplier * m)`
     - Investment growth is added by the hook (which computes per-scenario investment projections separately)
     - Return arrays for each scenario

   Actually, to keep this function self-contained and useful:

   ```typescript
   function projectPatrimonyScenarios(
     currentPatrimony: number,
     monthlyNetSavings: number,
     horizonMonths: number
   ): { optimista: number[]; base: number[]; pesimista: number[] }
   ```
   - For each scenario:
     - Month 0 = currentPatrimony
     - Month m = currentPatrimony + (monthlyNetSavings * savingsMultiplier * m)
   - Investment growth will be layered on top by the hook
