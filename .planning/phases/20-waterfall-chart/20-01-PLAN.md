  <behavior>
    ## Core behaviors to test:

    ### FLOW-01: Returns 5 bars with correct running totals
    - Input: salary 500000, 2 expenses totaling 100000 (all ARS, all variable), no investments
    - Expected: 5 bars named ["Ingresos", "Gastos Fijos", "Gastos Variables", "Inversiones", "Libre"]
    - Ingresos: barBottom=0, barTop=500000, amount=500000
    - Gastos Fijos: barBottom=500000, barTop=500000, amount=0 (none)
    - Gastos Variables: barBottom=400000, barTop=500000, amount=100000
    - Inversiones: barBottom=400000, barTop=400000, amount=0 (none)
    - Libre: barBottom=0, barTop=400000, amount=400000

    ### FLOW-02: recurringId classification
    - Input: 3 expenses -- 2 with recurringId ("netflix", "alquiler"), 1 without
    - Expected: Gastos Fijos amount = sum of 2 recurring, Gastos Variables amount = 1 non-recurring

    ### FLOW-03: Subcategory breakdown (top 5 + Otros)
    - Input: 7 variable expenses with different names/amounts
    - Expected: Gastos Variables subcategories has 6 items (top 5 by amount + "Otros" aggregating remaining 2)
    - Subcategories sorted by amount descending
    - "Otros" amount = sum of items ranked 6th and 7th

    ### FLOW-05: USD conversion per transaction
    - Input: 1 ARS expense (amount=50000) + 1 USD expense (amount=100, usdRate=1200)
    - Expected: total variable expenses = 50000 + (100 * 1200) = 170000
    - The USD expense's usdRate is used, NOT a global rate

    ### Investment filtering
    - Input: 2 investments each with movements in the month
      - Investment A: 1 aporte (50000, isInitial=false), 1 aporte (30000, isInitial=true -- excluded)
      - Investment B: 1 aporte (20000), 1 retiro (10000)
    - Expected: Net investment = 50000 + 20000 - 10000 = 60000 (isInitial excluded per D-10)

    ### Edge cases
    - Empty month (no expenses, no income): all bars have amount=0, Libre=0
    - Negative libre (expenses > income): Libre bar has barBottom=negative, barTop=0
    - Extra incomes included in Ingresos total (including USD extraIncomes converted)
    - Month filtering respects viewMode and payDay via getFilterDateRange
  </behavior>
  <implementation>
    Create `lib/projection/waterfall.ts` with:

    1. Type definitions:
       - `SubcategoryItem { name: string; amount: number; }`
       - `WaterfallBar { name: string; barBottom: number; barTop: number; amount: number; fill: string; subcategories: SubcategoryItem[]; }`
       - `WaterfallInput { expenses: Expense[]; investments: Investment[]; salaryAmount: number; extraIncomes: ExtraIncome[]; selectedMonth: string; viewMode: ViewMode; payDay: number; }`
       - NOTE: `salaryAmount` is the resolved salary from `getSalaryForMonth()` — it does NOT include aguinaldo. Aguinaldo is excluded from the waterfall because it's a semi-annual bonus, not monthly flow. The Ingresos bar = salaryAmount + converted extraIncomes.

    2. Helper functions:
       - `toArs(expense): number` -- converts to ARS using per-transaction usdRate
       - `toArsIncome(income): number` -- same for ExtraIncome
       - `classifyExpenses(expenses): { fixed: Expense[]; variable: Expense[] }` -- by recurringId presence
       - `buildSubcategories(items, getName, getAmount): SubcategoryItem[]` -- top 5 by amount + "Otros"

    3. Main `computeWaterfallData(input: WaterfallInput): WaterfallBar[]`:
       - Use getFilterDateRange to scope date range
       - Filter expenses within date range
       - Classify filtered expenses into fixed/variable
       - Convert all to ARS
       - Aggregate investment movements (exclude isInitial, net aportes-retiros within date range)
       - Convert USD investment amounts similarly
       - Build 5 bars with running total arithmetic
       - Compute subcategories for each bar

    4. Color constants:
       - WATERFALL_COLORS = { ingresos: "#22c55e", gastosFijos: "#ef4444", gastosVariables: "#f97316", inversiones: "#3b82f6", libre: "#10b981" }
