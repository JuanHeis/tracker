Create MANUAL.md at project root. Written entirely in Spanish using imperative/instructional tone ("Presiona", "Ingresa", "Selecciona"). Use the exact UI terminology from the app — never deviate (e.g., "Ingreso fijo" not "salario", "Patrimonio Total" not "valor neto").

Structure the manual with these sections in order:

1. **Introduccion** — What the app does, core value (reflejar la realidad financiera exacta), data stored in localStorage (no server).

2. **Primer Uso: Setup Wizard** — Document the full wizard flow:
   - Step 0 (Bienvenida): Two options — "Configurar desde cero" and "Importar backup existente"
   - Step 1 (Saldo ARS): Enter saldo liquido en ARS
   - Step 2 (Saldo USD): Enter USD balance + cotizacion (this step is skippable with "Omitir" or similar)
   - Step 3 (Ingreso Fijo): Enter ingreso fijo, tipo de empleo (dependiente/independiente), dia de cobro (skippable)
   - Step 4 (Inversiones): Add existing investments one by one with tipo, nombre, monto, moneda — loop to add more (skippable)
   - Step 5 (Resumen): Review all data and confirm — data saved atomically
   - Note: Draft saved in sessionStorage; closing tab loses draft
   - Note: Steps 2-4 can be skipped and configured later

3. **Pantalla Principal** — Describe the three top cards:
   - Resumen del Mes: Ingreso fijo, Otros ingresos, Aguinaldo, Total gastos, Aportes inversiones, Disponible
   - Patrimonio Total: Liquido ARS + Liquido USD (converted at global cotizacion) + Inversiones activas
   - Configuracion card (brief, full details in its own section)
   - Month/year selector and view mode toggle (Periodo vs Mes)
   - Bottom floating taskbar with 8 tabs

4. **Gastos** — How to add expense (description, amount, category, date), installments (cuotas) splitting across months, edit/delete expenses, categories.

5. **Ingresos** — Ingreso fijo (configured in Configuracion), Otros ingresos (add per month), Aguinaldo auto-calculated for dependiente (50% best salary of semester in June/December), two view modes: Periodo (dia de cobro to dia de cobro) vs Mes (calendar month), "Pendiente de cobro" indicator.

6. **Inversiones** — Create investment account (nombre, tipo: FCI/Plazo Fijo/Crypto/Acciones, moneda base), currency enforcement (Crypto=USD, Plazo Fijo=ARS), register aportes (reduces liquid) and retiros (increases liquid), update valor actual inline, finalizar investment (auto-retiro total + status change), rendimiento % and ganancia/perdida displayed, "Valor desactualizado" warning if >7 days since last update. Plazo Fijo auto-calculates value from rate and elapsed days.

7. **Charts** — Two charts available: Gastos por Mes (monthly expense comparison across the year) and Ingreso Fijo por Mes (salary comparison across the year). Both filter by selected year.

8. **Movimientos y Transferencias** — Transfers between own accounts (patrimonio unchanged), Compra USD from ARS balance (ARS decreases, USD increases), USD from cash no trackeado (with explicit origin), Ajustar saldo real (creates automatic adjustment entry to match declared balance).

9. **Gastos Recurrentes** — Define recurring expense (nombre, monto, categoria, frecuencia mensual), auto-generated each month, pause or cancel, mark as paid each month.

10. **Presupuestos** — Set monthly spending limit per category, visual progress bar (spent vs budget), alert when approaching or exceeding limit.

11. **Prestamos** — Register "le preste $X a [persona]" (loan given = asset in patrimonio) and "debo $X a [persona]" (debt = liability), mark as cobrado (money returns to liquid) or pagada (money leaves liquid).

12. **Configuracion** — Full section covering:
    - Tipo de empleo: dependiente (enables aguinaldo) vs independiente
    - Dia de cobro: configurable pay day
    - Cotizacion USD: global exchange rate for patrimonio calculation
    - Historial de sueldo: timeline of salary entries, add/edit/delete, increases apply from current month forward
    - Exportar datos: download all data as JSON file
    - Importar datos: upload JSON with validation, rejects corrupt/incompatible data, triggers full reload
    - Re-ejecutar wizard: resets ALL data (with confirmation dialog) and launches wizard as if first-time

13. **Moneda Dual (ARS/USD)** — Explain dual currency system: separate real balances, each transaction stores cotizacion at entry time, global cotizacion for patrimonio, compra USD flow, ganancia/perdida cambiaria (purchase rate vs current rate), retroactive cotizacion editing.

14. **Glosario de Terminos** — Table of all key terms: Ingreso fijo, Otros ingresos, Aguinaldo, Patrimonio Total, Liquido ARS/USD, Disponible, Pendiente de cobro, Gastos, Cuotas, Aportes, Retiros, Valor actual, Ganancia/perdida, Rendimiento %, Cotizacion USD, Dependiente/Independiente, Dia de cobro, Periodo/Mes, Recurrentes, Presupuestos, Prestamos, Transferencia, Ajustar saldo real, Re-ejecutar wizard.

Use numbered lists for step-by-step instructions. Use bold for UI element labels. Include notes/tips (prefixed with "> **Nota:**") for non-obvious behaviors like: cuotas date calculation, aguinaldo conditions, currency enforcement per investment type, atomic save behavior.

Target length: 250-400 lines. Comprehensive but not verbose.
