<objective>
Implement spec 260601-pmm â€” Purpose por Movement. Move the `purpose` classification
from the Investment level down to each InvestmentMovement (inheriting from the Investment
when absent), refine the Resultado del mes / Disponible calculation so that purpose is read
per-movement, and surface purpose in the UX (editor select, list chip, buckets-by-purpose view).

Purpose: The user has funds (e.g. SBS RTA PESOS) that mix layers â€” permanent savings plus
temporary card pre-funding. A single Investment-level label forces a wrong binary classification
and causes double-counting. Per-movement purpose models the real layering and fixes the
double-deduction bug.

Output: `purpose?` field on InvestmentMovement, a shared purpose constant module, a refined
month-metrics engine (per-movement evaluation), a pure buckets engine, and updated movement UX.
</objective>
